# Ayoub's Instructions — BKC Email Build

## Core Rule
NO SCRIPTS. No batching. No automation. One subagent per micro-task. Slow. Step by step.

---

## The Workflow

1. Take a copy file
2. Dissect it into sections (one subagent per section per file)
3. Choose components for each section (respecting variance rules)
4. Build the HTML email

---

## Email Structure (as described by Ayoub)

- **Header** = static logo (same on ALL emails, from Giuliano's raw file)
- **Hero** = headline above the fold, above image/component
- **Bridge section** = connects hero to next section
- **Body sections** = vary per email based on copy file structure
- **CTA** = sometimes early, sometimes late, sometimes text link
- **Footer** = static (from Giuliano's raw file)

### Key Concepts
- Headline must ALWAYS be above the fold
- Above the fold vs below the fold matters
- Bridge section can be: bullet points, standalone image, product showcase, SVG, or component
- CTA position varies per email

---

## Component Rules

- ONLY **product showcase** and **image showcase** are allowed to be reused
- Hero sections = NO reuse
- All other sections = NO reuse
- 4-8 components per email
- 4-6 images per email

---

## Variance (the whole point)

- Different structure per email = no desensitization
- Different components per email
- Different image choices per email
- If customer sees same structure twice → they stop opening/clicking/converting

---

## Copy Files

- Each has a different structure
- Creative suggestions in the files are suggestions, NOT dogma
- Must follow what each file says
- Each file contains: verbatim copy + creative suggestions + level calibration + visual direction
