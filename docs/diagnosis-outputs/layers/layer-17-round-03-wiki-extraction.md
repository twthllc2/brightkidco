## 3. FRAMEWORKS FROM WIKI (Applicable to BrightKidCo)

### 3.1 PAS — Problem-Agitate-Solve
**Source:** `SOP-copywriting-body-copy-frameworks.md` §1

| PAS Step | Generic | BrightKidCo Adaptation |
|----------|---------|----------------------|
| **Problem** | Identify pain point | Symptom recognition: "He talks about the potty but won't actually use it." |
| **Agitate** | Make problem feel worse | AVOID standard agitation. Instead: validate + absolve guilt. "That's not defiance. That's a signal-timing gap." |
| **Solve** | Present product | Mechanism explanation: "His nervous system processes the signal — just too late." THEN product as supporting layer. |

**Key difference:** Standard PAS agitates to create urgency. BrightKidCo VALIDATES to create trust. The "agitate" step becomes "validate + reframe."

### 3.2 AIDA — Attention-Interest-Desire-Action
**Source:** `SOP-copywriting-body-copy-frameworks.md` §2

| AIDA Step | Generic | BrightKidCo Adaptation |
|-----------|---------|----------------------|
| **Attention** | Bold claim, statistic | SPARK stat: "49% of autistic 4-5 year olds aren't trained." Or symptom hook. |
| **Interest** | Build engagement | Mechanism explanation: "Here's what's happening neurologically." |
| **Desire** | Benefits + social proof | Peer testimonials (messy, real). Kelly Mahler co-sign. BCBA validation. |
| **Action** | Clear CTA | "Try it for 60 days. By your judgment." Permission-not-to-buy energy. |

### 3.3 BAB — Before-After-Bridge
**Source:** `SOP-copywriting-body-copy-frameworks.md` §3

| BAB Step | Generic | BrightKidCo Adaptation |
|----------|---------|----------------------|
| **Before** | Problem state | Failed methods: "Sticker chart didn't work, did it? Here's why it couldn't." |
| **After** | Ideal state | "What if his nervous system could learn to feel the signal?" — never promise outcomes |
| **Bridge** | How product gets there | "The Body-Signal Learning Layer adds the missing piece." |

### 3.4 Storytelling Arc — Hook → Conflict → Resolution → CTA
**Source:** `SOP-copywriting-body-copy-frameworks.md` §4

Best for Lena's personal emails (Welcome E1-E2, Phase 2 validation):
- **Hook:** "Both my sons went through an Oh Crap attempt."
- **Conflict:** "It stressed all of us out."
- **Resolution:** "Here's what I learned: ADHD and autistic nervous systems don't send the same signals."
- **CTA:** "60 days to try it. By your judgment."

## 5. NEW RULES FROM WIKI (NOT in Client Files or Skills)

These rules exist in the wiki but are NOT documented in R2 (Skill Loading), L9 (Voice Rulebook), or client docs:

### 5.1 Voice Matrix Axes (HIGH VALUE)
**Source:** `SOP-copywriting-brand-voice-storytelling.md` §1.2, `brand-voice-definition.md`

The wiki's 4-axis voice matrix (Formal/Casual, Serious/Playful, Respectful/Irreverent, Practical/Aspirational) provides a structured way to DOCUMENT Lena's voice position. Client docs describe Lena's voice qualitatively but never map it to a formal axis system. This would make voice consistency auditable.

**Recommendation:** Adopt the 4-axis matrix for Lena voice documentation. Position: Casual(4), Serious(2), Respectful(1), Practical(3).

### 5.2 Brand Personality Spectrum (MEDIUM VALUE)
**Source:** `brand-voice-definition.md`

5-dimensional personality mapping (Sincerity, Excitement, Competence, Sophistication, Ruggedness). BrightKidCo = Sincerity-primary (60%) + Competence-secondary (30%). Provides a formal framework for quarterly voice audits.

**Recommendation:** Use for voice consistency scoring. Target VCS >85% on Sincerity dimension.

### 5.3 80/20 Education-to-Promotion Ratio (HIGH VALUE)
**Source:** `SOP-copywriting-brand-voice-storytelling.md` §5.1

"8 should provide value, 2 should directly promote per 10 emails." Client docs emphasize education-first but never quantify the ratio. This gives a concrete benchmark.

**Recommendation:** Adopt. BrightKidCo's mechanism-first approach already implies this ratio. Formalizing it prevents drift.

### 5.4 "So What?" Test (HIGH VALUE)
**Source:** `SOP-copywriting-body-copy-frameworks.md` §5

"Every sentence must earn its place. Ask 'So what?' — if it doesn't move the reader closer to action or pass a value test, cut it." Value tests: inform, build desire, overcome objection, create urgency, establish credibility.

**Recommendation:** Adopt as quality gate for all BrightKidCo emails. Especially critical for Phase 3 parents (decision fatigue at chronic levels).

### 5.5 Bold Reading Path Technique (MEDIUM VALUE)
**Source:** `SOP-copywriting-body-copy-frameworks.md` §10.2

"Write the email, bold key phrases, then read only bold text. If bold-only version doesn't tell the story, bold choices are wrong."

**Recommendation:** Adopt for email QA. Especially useful for mobile-first readers (60-70% of opens).

### 5.6 Reply-to-This-Email Strategy (MEDIUM VALUE)
**Source:** `SOP-copywriting-brand-voice-storytelling.md` §10

"Replies are the strongest positive engagement signal for mailbox providers." Strategies: ask questions, offer help, request feedback, make it personal.

**Recommendation:** Add "Reply to this email — Lena reads every response" to footer. Klaviyo reply handling SOP needed.

### 5.7 Objection-CTA Sandwich (MEDIUM VALUE)
**Source:** `SOP-copywriting-body-copy-frameworks.md` §8.3

Structure: [Benefit/Hook] → [Price/Value justification or guarantee] → [CTA] → [PS: re-state guarantee, urgency, or social proof]

**Recommendation:** Adopt for cart abandonment and conversion-stage emails. Guarantee must appear BEFORE CTA, not after.

### 5.8 PS Line Rules (MEDIUM VALUE)
**Source:** `SOP-copywriting-body-copy-frameworks.md` §8.3

- PS = second most-read part of email (after subject line)
- One PS only (two looks cluttered) — **CONFLICTS with L9 which allows 2-line PS**
- Keep shorter than main body
- Bold PS label for scannability

**Conflict with L9:** L9 allows "max 2 lines P.S." Wiki says "one PS only." Resolution: L9's "2 lines" means 2 sentences within ONE PS block, not two separate PS sections. Compatible.

### 5.9 Content Pillars Framework (LOW VALUE)
**Source:** `SOP-copywriting-brand-voice-storytelling.md` §6

3-5 core topics the brand "owns." For BrightKidCo: (1) Interoception Science, (2) Failed Methods Explained, (3) Parent Stories, (4) Body-Signal Learning, (5) Autism Identity.

**Recommendation:** Useful for content calendar planning. Not directly tonality but prevents topic drift.

### 5.10 Voice vs Tone Distinction (LOW VALUE)
**Source:** `SOP-copywriting-brand-voice-storytelling.md` §1.1

"Voice = consistent personality, never changes. Tone = emotional expression in specific context, changes with situation." Client docs blur this distinction. Formalizing it helps writers understand: Lena's VOICE never changes, but her TONE shifts from "hope-forward" (L1 Welcome) to "dignity-first" (L3 Winback).

**Recommendation:** Adopt terminology. Voice = Lena's personality. Tone = phase/level-specific calibration.

### 5.11 Subject Line Formulas (MEDIUM VALUE)
**Source:** `SOP-copywriting-subject-line-engineering.md` §2

7 formulas: Curiosity Gap, Direct Benefit, Urgency/Scarcity, Personalization, Question, How-To, Social Proof.

**BrightKidCo applicability:**
- Curiosity Gap: USE — "What his OT didn't tell you about potty training"
- Direct Benefit: ADAPT — "60 days to try the body-signal approach"
- Urgency/Scarcity: L1 ONLY — "Pre-K in 4 months — here's the plan"
- Personalization: USE — [First Name] + symptom hook
- Question: USE — "Still in pull-ups? Here's why it's not your fault."
- How-To: USE — "How interoception explains why sticker charts failed"
- Social Proof: USE — "16,511 parents rated this 4.9 stars"

### 5.12 Sentence Length by Funnel Stage (LOW VALUE)
**Source:** `brand-voice-definition.md` §Tone Guidelines Matrix

- TOF (Awareness): 8-12 words per sentence
- MOF (Consideration): 14-18 words per sentence
- BOF (Conversion): Variable

**Recommendation:** Useful as guideline. Phase 1 = shorter sentences (education). Phase 3 = longer (mechanism density).

### 5.13 Vocabulary Tier by Audience (LOW VALUE)
**Source:** `brand-voice-definition.md` §Tone Guidelines Matrix

- Tier 1 = Grade 5 reading level for broad/TOF audiences
- Tier 2 = Grade 8 for retargeting/MOF

**Recommendation:** BrightKidCo should default to Tier 1 (accessible). Mechanism explanations should stay at Grade 5-6 level even when citing Kelly Mahler.

## 7. WIKI FRAMEWORK ADAPTATIONS FOR BRIGHTKIDCO

### 7.1 Modified PAS for BrightKidCo
```
Standard PAS:    Problem → Agitate → Solve
BrightKidCo PAS: Symptom → Validate → Mechanism → Permission
```
The "Agitate" step is REPLACED with "Validate" because BrightKidCo parents are already in pain. Agitation would trigger shame. Validation builds trust.

### 7.2 Modified AIDA for BrightKidCo
```
Standard AIDA:    Attention → Interest → Desire → Action
BrightKidCo AIDA: Symptom Hook → Mechanism → Peer Proof → Permission CTA
```
"Desire" becomes "Peer Proof" (testimonials, Kelly Mahler). "Action" becomes "Permission CTA" (60 days, your judgment).

### 7.3 Modified BAB for BrightKidCo
```
Standard BAB:    Before → After → Bridge
BrightKidCo BAB: Failed Methods → What If → Body-Signal Layer
```
"Before" names specific failed methods. "After" is a QUESTION, not a promise. "Bridge" is the mechanism, not the product.

### 7.4 Lena's Sentence Templates (from wiki frameworks)
These templates combine wiki frameworks with Lena's voice:

**Observation (Hook):**
- "If your [age]-year-old isn't [behavior] — that's completely normal for kids with sensory differences."

**Validation (Problem → Validate):**
- "It wasn't you. It wasn't your child. It was the method."

**Mechanism (Interest → Desire):**
- "Here's what's happening neurologically: [mechanism]."
- "Kelly Mahler's research on interoception explains why: [explanation]."

**Permission (Action):**
- "Try it for 60 days. By your judgment."
- "We're not asking you to hope. We're asking you to understand the mechanism."
