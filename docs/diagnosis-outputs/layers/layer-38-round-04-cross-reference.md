## 4. IDENTIFIED INCONSISTENCIES

### 4.1 Brand-Yellow Hex Code (MINOR)

**Issue:** L4 §1 defines Brand-Yellow as `#F5C84B`. L29 §1.1 lists it as `#F5C84B` / `#F5CB5C` (two values).
**Impact:** Low — the two hex values are visually near-identical. But for code/CSS implementation, only one should be used.
**Recommendation:** Standardize on `#F5C84B` across all layers. Remove `#F5CB5C` from L29 or document it as an acceptable alternate.
**Severity:** 🟡 LOW

### 4.2 L38 R1 (Client Files) Output Missing

**Issue:** The task spec references `/root/projects/brightkidco/outputs/diagnosis/layer-38-round-01-client-files.md` as an input, but this file does not exist. No Layer 38 round outputs exist in the diagnosis directory.
**Impact:** Medium — brand constants had to be reconstructed from cross-layer references in L4, L29, and L33 rather than from a dedicated L38 client file extraction.
**Recommendation:** Create L38 R1 by extracting brand constants from DOC1-DOC6 client files. This would serve as the authoritative single source of truth for all brand constants.
**Severity:** 🟡 MEDIUM

### 4.3 L09 File Naming Mismatch

**Issue:** The task spec references `layer-09-voice.md` but the actual file is `layer-09-voice-lena.md`.
**Impact:** Low — file was found via search. But downstream automation that parses file paths literally would fail.
**Recommendation:** Update task specs to use the correct filename, or create a symlink.
**Severity:** 🟢 LOW

### 4.4 Parent Trust Count Not in L4 or L9

**Issue:** "100,000+ parents trust BrightKidCo" appears in L29 §1.5 (footer) and L33 §9.6 (footer) but is absent from L4 (Brand Constants) and L9 (Voice).
**Impact:** Low — the stat is used only in the footer microcopy. But its absence from L4 means it could be forgotten in non-email contexts.
**Recommendation:** Add "100,000+ parents" to L4 §1 Brand Constants for completeness.
**Severity:** 🟢 LOW

### 4.5 Offer "Save" Tags Only in L29

**Issue:** The "Save 40% TODAY" and "Save 60% TODAY" tags are defined only in L29 §1.7. L33 §6.2 (S5) defines the pricing table without these save tags.
**Impact:** Medium — email creators using L33 as their primary reference will miss the save tags. L29 is the visual calibration layer, not the format layer, so content creators may not look there.
**Recommendation:** Add save tags to L33 §6.2 pricing table for consistency.
**Severity:** 🟡 MEDIUM

### 4.6 Cream Alt Color Only in L29

**Issue:** `#FAF5E8` (Cream Alt) appears only in L29 §1.1. It is absent from L4's color list and from any other layer.
**Impact:** Low — it's a subtle variant of Cream Card. But designers need to know it exists.
**Recommendation:** Add Cream Alt to L4 §1 if it's an intended brand constant, or remove it from L29 if it's just a design exploration.
**Severity:** 🟢 LOW

## 7. RECOMMENDATIONS FOR ALIGNMENT

### 7.1 Immediate (Pre-Launch)

1. **Standardize Brand-Yellow hex to `#F5C84B`.** Remove `#F5CB5C` variant from L29 or document as alternate. One hex, one constant.

2. **Add "Save" tags to L33 pricing table.** L33 §6.2 (S5) should include "Save 40% TODAY" and "Save 60% TODAY" alongside the pricing, since L33 is the primary format reference for email creators.

3. **Create L38 R1 (Client Files).** Extract brand constants from DOC1-DOC6 into a single authoritative L38 output. This becomes the single source of truth that all other layers reference.

### 7.2 Short-Term (Launch + 30 Days)

4. **Add "100,000+ parents" to L4 Brand Constants.** This stat appears in footer (L29, L33) but not in the avatar levels document where brand constants are listed.

5. **Decide on Cream Alt (`#FAF5E8`).** Either promote it to a documented brand constant across layers, or remove it from L29 as a design exploration that didn't make the cut.

6. **Create a brand constants single-source-of-truth file.** A standalone document (or L38 final synthesis) that lists every brand constant with its canonical value, so downstream layers don't need to cross-reference each other.

### 7.3 Longer-Term

7. **Automate cross-layer consistency checks.** Build a validation script that compares brand constant values across L4, L9, L29, and L33 outputs and flags drift.

8. **Version-lock brand constants.** When a constant changes (e.g., review count updates from 16,511 to a new number), propagate the change to ALL dependent layers simultaneously.
