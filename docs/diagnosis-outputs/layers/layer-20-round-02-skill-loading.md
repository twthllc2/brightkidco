## Skill 1: Copywriting Brand Voice & Storytelling

**Source:** `/root/.hermes/skills/email-marketing/strategy-analytics/copywriting-brand-voice-storytelling/SKILL.md`

### Framework 1.1 — Voice Axes (4-Axis Model)

The Voice Matrix defines brand personality across four axes. For cross-level writing, the axes determine how conditional language is calibrated:

| Axis | Cross-Level Application |
|------|------------------------|
| Formal vs Casual | "Whether" patterns in formal voice: "Whether you seek..." vs casual: "Whether you're into..." |
| Serious vs Playful | Serious conditional: "If this matters to you..." vs playful: "If you're the kind of person who..." |
| Respectful vs Irreverent | Respectful: "Whether you prefer..." vs irreverent: "Whether you're a rebel or a rule-follower..." |
| Practical vs Aspirational | Practical: "If you need X..." vs aspirational: "If you dream of X..." |

**Application to Layer 20:** The "Whether...Or...If..." pattern must slide across all four axes depending on audience segment. A single conditional statement can be rewritten in 16+ variations (4 axes × 2 poles × 2 directions).

### Framework 1.2 — Segment-Specific Tone Adjustments

Voice adaptation by segment directly maps to conditional language targeting:

| Segment | Conditional Framing |
|---------|---------------------|
| VIP / High LTV | "Whether you're celebrating your 10th order or your first..." |
| New Subscribers | "If you're just getting started, here's what matters..." |
| Lapsed / Winback | "Whether it's been a week or a year, we saved your spot..." |
| Abandoned Cart | "If you're still deciding, we made it easier..." |
| Post-Purchase | "Whether this was a gift or a treat for yourself..." |
| Cold / Unengaged | "If this isn't relevant anymore, no hard feelings..." |

**Key Insight:** The relationship ladder (New → First Purchase → Repeat → VIP → Advocate) dictates the conditional depth. New subscribers get simple "if/then" patterns; VIPs get layered "whether...or...and even if..." patterns.

### Framework 1.3 — The Relationship Ladder for Conditional Depth

```
NEW SUBSCRIBER:   "If you're new here..." (simple binary)
       ↓
FIRST PURCHASE:   "Whether you loved it or have questions..." (binary with empathy)
       ↓
REPEAT BUYER:     "Whether you're restocking or exploring..." (multi-path)
       ↓
VIP:              "Whether you've tried everything or just this..." (insider knowledge)
       ↓
ADVOCATE:         "Whether you've told 1 friend or 100..." (partnership framing)
```

### Framework 1.4 — Vocabulary Control (Always/Never/Sometimes)

Conditional language must respect brand vocabulary constraints:

- **Always use:** Inclusive, empowering, specific conditional words
- **Never use:** Exclusionary, assumptive, or pressure-loaded conditionals
- **Safe conditional connectors:** "Whether," "If," "Even if," "No matter if," "Regardless of whether"
- **Avoid:** "But if you don't..." (negative conditional), "Only if..." (exclusionary)

### Framework 1.5 — Objection Handling as Conditional Structure

Objection handling naturally produces conditional language:

| Objection | Conditional Rewrite |
|-----------|---------------------|
| Price too high | "Whether you're comparing cost-per-use or upfront price..." |
| Timing not right | "If now isn't the right time, we'll be here when it is..." |
| Better options exist | "If you've tried others and they didn't work..." |
| Trust (will it work?) | "Whether you've been burned before or this is your first try..." |
| Commitment fear | "If you're not sure, try it risk-free..." |

## Skill 2: Copywriting Body Copy Frameworks

**Source:** `/root/.hermes/skills/email-marketing/strategy-analytics/copywriting-body-copy-frameworks/SKILL.md`

### Framework 2.1 — PAS (Problem-Agitate-Solve) with Conditional Language

PAS maps directly to the "Whether...Or...If..." pattern:

1. **Problem (If):** "If you've ever struggled with [problem]..."
2. **Agitate (Whether...Or):** "Whether you've tried everything or nothing at all..."
3. **Solve (Then):** "...here's what actually works."

**Conditional PAS Template:**
```
If [problem resonates with reader]...
Whether you've [tried solution A] or [tried solution B]...
[Product/Brand] is the [solution type] that [key benefit].
```

### Framework 2.2 — AIDA with Inclusive Framing

AIDA adapted for cross-level writing:

1. **Attention:** "If you're reading this, something caught your eye."
2. **Interest:** "Whether you're a beginner or an expert..."
3. **Desire:** "Imagine if [benefit] were already yours."
4. **Action:** "If you're ready, [CTA]."

### Framework 2.3 — BAB (Before-After-Bridge) as Conditional Transformation

BAB naturally supports "Whether...Or...If..." because it frames transformation as conditional:

- **Before:** "If you're stuck in [current state]..."
- **After:** "Whether you believe it or not, [ideal state] is possible."
- **Bridge:** "Here's how [product] gets you there — no matter where you're starting from."

### Framework 2.4 — Storytelling Arc with Conditional Hooks

The Hook → Conflict → Resolution → CTA framework uses conditional language at each stage:

- **Hook:** "If you've ever [relatable experience]..."
- **Conflict:** "Whether you know it or not, [hidden problem] is costing you..."
- **Resolution:** "Even if [common objection], here's why it works..."
- **CTA:** "If this resonates, [action]."

### Framework 2.5 — The "So What?" Test for Conditional Copy

Every conditional statement must pass the "So What?" test:

| Conditional Statement | So What? Test | Result |
|----------------------|---------------|--------|
| "Whether you're a size 6 or 16..." | Does it build inclusivity? | ✅ Pass |
| "If you're interested..." | Does it create urgency? | ❌ Cut — too vague |
| "Whether it's morning or night..." | Does it show versatility? | ✅ Pass |
| "If you want to..." | Does it add value? | ❌ Cut — passive |

### Framework 2.6 — Tone Spectrum for Conditional Calibration

| Tone | Conditional Style |
|------|-------------------|
| Promotional | "If you act now..." / "Whether you shop today or tomorrow..." |
| Educational | "If you've wondered why..." / "Whether you're new to this or experienced..." |
| Personal | "If I'm being honest..." / "Whether you believe me or not..." |
| Urgent | "If you wait..." / "Whether you decide now or later, the price goes up..." |
| Celebratory | "If you're ready to celebrate..." / "Whether it's your first win or your fiftieth..." |
| Empathetic | "If you've been through a lot..." / "Whether you're ready or still healing..." |

## Skill 3: Copywriting Subject Line Engineering

**Source:** `/root/.hermes/skills/email-marketing/strategy-analytics/copywriting-subject-line-engineering/SKILL.md`

### Framework 3.1 — Question Formula as Conditional Gateway

The Question formula naturally produces conditional subject lines:

- "Still thinking it over?" → Implies "If you're still thinking..."
- "Ready for your next adventure?" → Implies "If you're ready..."
- "What if you could save 20%?" → Direct conditional framing
- "Need a gift idea?" → Implies "If you need..."
- "Forgetting something?" → Implies "If you forgot..."
- "What's your skincare routine missing?" → Implies "If your routine is incomplete..."

**Conditional Subject Line Templates:**
```
"If [pain point], you're not alone"
"Whether you're [type A] or [type B], this works"
"What if [desirable outcome] were easy?"
"If you've been waiting for [X], it's here"
"Whether you know it or not, [insight]"
```

### Framework 3.2 — Curiosity Gap with Conditional Framing

Curiosity gap subject lines work by implying conditional knowledge:

- "You won't believe what just happened..." → Implies "If you knew..."
- "The one product our CEO couldn't stop talking about" → Implies "If you're curious..."
- "We tried something new — and it worked" → Implies "If you've been skeptical..."
- "The email we almost didn't send" → Implies "If we hadn't..."

### Framework 3.3 — Personalization as Conditional Branching

Personalization in subject lines is inherently conditional:

```
[IF subscriber_data = "first_name"]:
  "[Name], your cart is waiting"
[ELSE]:
  "Your cart is waiting"
```

This maps to the "Whether...Or...If..." pattern:
- "Whether you're [Name] or just discovering us..."
- "If you're [Name], we picked these just for you"

### Framework 3.4 — How-To Formula for Conditional Education

How-to subject lines frame solutions as conditionals:

- "How to double your email revenue" → "If you want to double..."
- "How to style our new collection" → "If you're wondering how..."
- "3 ways to wear the same dress" → "Whether you're dressing up or down..."

### Framework 3.5 — Social Proof as Conditional Validation

Social proof subject lines validate through conditional framing:

- "10,000 people bought this last week" → "If 10,000 people trusted this..."
- "Join 50,000+ happy customers" → "Whether you're #50,001 or #100,000..."
- "Rated 4.8 stars by our community" → "If ratings matter to you..."

The Welcome Series (7 emails) maps to progressive conditional depth:

| 6: Brand story | "Whether you care about origins or just results..." |
| 7: First month recap | "If this month is any indication..." |

### Framework 4.2 — Cart Abandonment as Conditional Urgency

The 3-email abandonment sequence uses escalating conditional urgency:

1. **1h:** "If you're still thinking it over..." (soft conditional)
2. **24h:** "Whether you forgot or got busy..." (empathetic conditional)
3. **72h:** "If you want these before they sell out..." (urgent conditional)

### Framework 4.3 — PAS with Ethical Conditional Framing

From the Copy skill's PAS framework, adapted for inclusive language:

- **Problem:** "If you've been struggling with [X]..." (validates experience)
- **Agitate:** "Whether you've tried [solution A] or [solution B]..." (acknowledges effort)
- **Solve:** "Even if nothing has worked before..." (inclusive of failures)

### Framework 4.4 — BAB for Aspirational Conditionals

Before-After-Bridge with conditional framing:

- **Before:** "If you're stuck where you are..."
- **After:** "Imagine if [desired state] were already real."
- **Bridge:** "Whether you believe it's possible or not, here's how."

### Framework 4.5 — AI-ism Banned List (Conditional Language Filter)

Banned words that contaminate conditional copy:

**NEVER use in conditional statements:**
- "If you want to *dive into*..." → Use "If you want to explore..."
- "Whether you're looking for a *game-changer*..." → Use "Whether you're looking for something that works..."
- "If you're ready to *unlock*..." → Use "If you're ready to discover..."
- "Whether you're *leveraging*..." → Use "Whether you're using..."

### Framework 4.6 — Quality Gate for Conditional Copy

Every conditional statement must pass:

- [ ] No AI-isms from banned list
- [ ] Brand voice matches brief
- [ ] Inclusive (doesn't exclude any audience segment)
- [ ] Specific (conditional leads to a clear outcome)
- [ ] Natural flow when read aloud
- [ ] Would you engage with this if you received it?

## Skill 5: Content Strategy

**Source:** `/root/.hermes/skills/content/SKILL.md`

### Framework 5.1 — 4-Pillar System as Conditional Content Map

The TOFU/MOFU/BOFU/Retention model maps to conditional audience targeting:

| Funnel Stage | Conditional Framing |
|-------------|---------------------|
| TOFU (40%) | "If you're just discovering [topic]..." / "Whether you're new to this or curious..." |
| MOFU (30%) | "If you've been researching [topic]..." / "Whether you've tried X or Y..." |
| BOFU (20%) | "If you're ready to decide..." / "Whether you choose us or not, here's what to look for..." |
| Retention (10%) | "Whether you've been with us 1 month or 1 year..." / "If you loved [product]..." |

### Framework 5.2 — Topic Clusters for Multi-Audience Content

Pillar-cluster architecture naturally supports "Whether...Or...If..." patterns:

- **Pillar page:** "Everything you need to know about [topic]" → "Whether you're a beginner or expert..."
- **Cluster pages:** Specific subtopics → "If you're interested in [specific aspect]..."
- **Cross-links:** "If you found this helpful, you might also like..." / "Whether you prefer [A] or [B]..."

### Framework 5.3 — Intent Classification for Conditional Targeting

| Intent | Conditional Style |
|--------|-------------------|
| Informational | "If you're wondering how/why/what..." |
| Commercial | "If you're comparing options..." / "Whether you're looking at [A] or [B]..." |
| Transactional | "If you're ready to buy..." / "Whether you want [option 1] or [option 2]..." |
| Navigational | "If you're looking for [brand]..." |

### Framework 5.4 — Writing Standards for Inclusive Language

Content skill writing standards applicable to conditional copy:

- **Paragraphs 3-4 sentences:** Conditional statements should be concise
- **Active voice (90%):** "If you choose X" not "If X is chosen by you"
- **Second person:** "If you..." not "If one..." or "If customers..."
- **Data cited:** Conditional claims backed by evidence ("If you're among the 73% who...")

### Framework 5.5 — Quality Pass for Conditional Content

From the Content skill's quality standards:

- Facts checked (conditional claims are accurate)
- AI-isms removed (no "game-changer" in conditionals)
- Fluff cut (every conditional earns its place)
- Read aloud test (conditional flow is natural)
- Better than top 3 results (conditional framing is more inclusive than competitors)
- Would you share this? (conditional copy feels authentic, not manipulative)

## Practical Application to Layer 20

### The "Whether...Or...If..." Formula for General Fallback

Layer 20's General Fallback uses conditional language to reach every audience member without assuming their state:

**Core Pattern:**
```
Whether you're [audience type A] or [audience type B]...
If you've [common experience]...
[Universal value proposition with specific proof]
[CTA that works for all paths]
```

**Example — BrightKidCo Diagnosis Flow:**
```
Whether you're a first-time parent or raising your third...
If you've ever wondered whether your child's behavior is "normal"...
Our diagnostic framework gives you clarity — backed by 10,000+ assessments.
[Get Started]
```

### Conditional Language Rules for Layer 20

1. **Never assume audience state.** Use "Whether" to acknowledge multiple states exist.
2. **Lead with inclusion.** "Whether A or B" before "If C" — broader first, specific second.
3. **Match conditional depth to relationship stage.** Simple for new, complex for engaged.
4. **Every conditional must pass the "So What?" test.** If it doesn't add value, cut it.
5. **Respect vocabulary constraints.** No banned words, no AI-isms.
6. **Test conditional statements read aloud.** They should sound like a real person talking.
7. **Use the voice axes to calibrate tone.** The same conditional can be formal or casual.
