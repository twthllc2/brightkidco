### 1. WELCOME FLOW (Flow 01) — 32 files, 8 emails, 8 SKELETONS

**Note:** This is the only flow with level-specific variants (L1, L2, L3, GF = 4 copies per email). Each email position has a distinct purpose and section structure.

#### Email 1 (Day 0) — Skeleton: "Education Hook"
```
Sections: Hook → Validation → Mechanism → CTA
```
- **Purpose:** Immediate validation + mechanism teaser
- **Emotional Arc:** Mirror confusion → Validate → Mechanism → Curiosity
- **Key Components:** Single-column text, parent quote block (Cream Card), stat callout box (Yellow accent), CTA button
- **NO product, NO testimonial, NO guarantee** — pure education
- **Visual Signature:** Signal-wave divider between Hook and Validation, stat callout box with Yellow top border

#### Email 2 (Day 2) — Skeleton: "Failed Methods Deep Dive"
```
L1/GF: Hook → Failed Methods Recognition → Mechanism Deep → CTA (4 sections)
L2/L3: Hook → Failed Methods → Validation → Mechanism Deep → Social Proof → Product → Guarantee → CTA (8 sections)
```
- **Purpose:** Name the methods that failed, introduce mechanism as the missing piece
- **Key Differentiator from E1:** Longer mechanism section, method-specific validation
- **Level Variation:** L1/GF stay education-only (4 sections); L2/L3 expand to full funnel (8 sections) with product intro
- **Visual Signature:** Failed-methods recognition block, deeper mechanism explanation

#### Email 3 (Day 5) — Skeleton: "Product Introduction"
```
Sections: Hook → Validation → Mechanism → Testimonial → Product → Guarantee → CTA
```
- **Purpose:** First product introduction with testimonial proof
- **Key Components:** Three-Family Testimonial pattern, 3-Layer System visual, Guarantee panel
- **Visual Signature:** Product layer stack illustration, testimonial card blocks, guarantee callout

#### Email 4 (Day 7) — Skeleton: "Three-Family Proof"
```
Sections: Hook → Validation → Mechanism → Testimonial (Three-Family Pattern) → Product → Guarantee → CTA
```
- **Purpose:** Social proof through three distinct family stories (L1/L2/L3)
- **Key Differentiator from E3:** Three-Family Testimonial is the PRIMARY section (not just one testimonial)
- **Visual Signature:** Three side-by-side or stacked testimonial cards, each with level-specific story

#### Email 5 (Day 10) — Skeleton: "Objection Handling"
```
L1: Hook → Validation (Objection Naming) → Reframe → Social Proof → Product → Guarantee → CTA (7 sections)
L2: Hook → Objection Naming → Reframe → Mechanism → CTA (5 sections, no product)
L3: Hook → Validation → Reframe → Mechanism → Social Proof → Product → Guarantee → CTA (8 sections)
GF: Hook → Objection Naming → Reframe → Mechanism → Social Proof → Product → Guarantee → CTA (8 sections)
```
- **Purpose:** Address "I've tried everything" objection
- **Key Components:** Reframe section (unique to this email), objection-naming block
- **Visual Signature:** Reframe section with mechanism explanation, objection-naming callout

#### Email 6 (Day 14) — Skeleton: "Pull-Up Reframe"
```
L1/L2: Hook → Validation → Mechanism (Pull-Up Reframe) → Social Proof → Product → Guarantee → CTA (7 sections)
L3: Hook → Gentle Pull-Up Reframe → Mechanism → Guarantee → CTA (5 sections)
```
- **Purpose:** Reframe pull-ups as the barrier, not the solution
- **Key Components:** Pull-Up Reframe section (unique to this email), comparison visual
- **Visual Signature:** Pull-up vs Body-Signal comparison, reframe callout block

#### Email 7 (Day 17) — Skeleton: "Deep Mechanism + Mini-Wins"
```
L1/L2/GF: Hook → Validation → Deep Mechanism → Mini-Win Testimonials → Product → Guarantee → CTA (7 sections)
L3: Hook → Mechanism → Guarantee → CTA (4 sections, minimal)
```
- **Purpose:** Deeper mechanism education with mini-win testimonials
- **Key Components:** Extended mechanism section, mini-win testimonial pattern
- **Visual Signature:** Deep mechanism callout, mini-win testimonial cards (shorter, more specific)

#### Email 8 (Day 21) — Skeleton: "Final Offer / Soft Close"
```
L1: Hook → Validation → Recap → Social Proof → Product → Guarantee → CTA → PS (8 sections)
L2: Hook → Validation → Social Proof → Guarantee → CTA (5 sections)
L3/GF: Hook → Recap → CTA (3 sections, minimal)
```
- **Purpose:** Final invitation, soft close, no urgency
- **Key Components:** Recap section (unique to this email), PS line, permission-frame CTA
- **Visual Signature:** Recap block with 3 bullet columns, guarantee panel, PS line below sign-off

## KEY INSIGHTS

### 1. Cross-Level Flows Are Simpler
Most post-purchase flows (PP-Education, PP-Direct Upsell, PP-Extended Upsell, PP-Mid Check-In, Replenish A/B/C) use a SINGLE skeleton per flow because they serve all 4 levels with one copy. The visual layout doesn't change — only the copy calibration shifts.

### 2. Welcome and Winback Are Complex
These are the only flows with level-specific variants (L1, L2, L3, GF). Different levels have different section structures for the same email position, creating MORE skeletons. Welcome needs 8 skeletons (one per email position, with level variants sharing the same skeleton). Winback needs 3 skeletons per path.

### 3. The "Whether...Or...If..." Pattern Is Universal
Cross-level flows universally use the "Whether your child is 3 or 10, verbal or non-verbal..." opener pattern. This is a consistent visual element across all cross-level skeletons.

### 4. Transactional Emails Are the Most Diverse
The 4 transactional emails have 3 distinct skeletons because each serves a different purpose (order confirmation vs shipping vs delivery vs arrival). They're the most structurally varied group.

### 5. FAQ Library Has 2 Clear Patterns
Single-objection emails (E1, E3, E5) and multi-objection emails (E2, E4, E6) have fundamentally different visual structures. The multi-objection pattern uses a three-section parallel layout.

### 6. Replenishment Flows Share One Skeleton
Replenish A, B, and C all use the same visual pattern despite targeting different buyer segments (1-pair, 3+3, 5+5). The skeleton is universal — only the product offer changes.
