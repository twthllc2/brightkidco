## 1. Flow 17b — FAQ Objection L2/L3

### 1.1 Level Structure Overview

| Level | Emails | Coverage |
|:-----:|:------:|----------|
| L2 | 1 email | BCBA conflict — interoception vs behavioral methods |
| L3 | 6 emails | D1→D4 (resignation, hope-risk, too-old, brand-unknown), C1→C3 (financial, sensory, GI), B1→B4 (underwear, BCBA, sensory refusal, withholding), Summary |

The structure asymmetry is notable: L2 has a single email while L3 has a full 6-email sequence. The README states the flow has 6 emails × 4 level variants, but only L2-email-01 exists for L2 — no E2–E6 have been built. The L2-email-01 covers the B2 (BCBA conflict) objection from the README's E5 slot, but there is no L2 treatment of D1/D2 (E1), D3/C3 (E2), B1/B3 (E3), C1/C2 (E4), or the summary (E6).

**Assessment:** L2 is incomplete — 1 of 6 required emails. The single existing email is strong, but the L2 subscriber will receive only one FAQ email compared to 6 for L3. This creates a significant experience gap.

**Recommendation:** Build L2 variants for E1–E4 and E6 at minimum, following the same objection mapping as the other levels. The L2 parent profile (verbal child, working with therapists, may have some interoception but not enough) needs tailored messaging for each objection, not just the BCBA one.

### 1.2 L2-email-01 — BCBA Conflict (B2 Objection)

**Subject: "Your BCBA did everything right — this adds one thing she couldn't"**

**Assessment: EXCELLENT in isolation.** This is the strongest single email in the entire FAQ Objection set for its target audience.

**Strengths:**
- Opens by acknowledging the BCBA work without diminishing it — "Your BCBA has done everything right" is disarming and respectful
- Frames the problem as neurological, not behavioral — "It's not behavioral. It's neurological." — which removes the parent's self-blame
- Cites real research (Kelly Mahler, Barmpagiannis et al. 2025) at a level an L2 parent would find helpful but not overwhelming
- Offers the BCBA a concrete bridge: "BSL isn't a replacement — it's a sensory bridge" — which gives the parent language to use with their therapist
- The P.S. offering a one-page clinical summary for the BCBA is a brilliant trust-building mechanism
- Subject line options are well-constructed and audience-appropriate

**Weaknesses:**
- This is a single email handling what the README maps to E5 (Day 31, B2 objection). The L2 parent is missing the lower-funnel objections (D1: tried everything, D3: hope-risk, C1: brand unknown) that an L2 parent would also face
- No subject line option that mentions the BCBA clinical summary offer — that's a potentially high-CTR hook for this audience
- The CTA ("See the mechanism that bridges what your BCBA can't reach") is well-targeted but has no secondary CTA

**Recommendations:**
1. Build the remaining 5 L2 emails — L2 parents need the full objection sequence
2. Add a subject line variation highlighting the BCBA summary offer: "A one-page clinical summary for your BCBA — reply and I'll send it"
3. Add a second CTA: link to the clinical summary landing page or a relevant research page alongside the main product CTA
4. Consider whether L2 E1 should also address the D1 (tried everything) objection with a different first email, or combine L2 into a shorter 3-email sequence — the current 1-out-of-6 is neither

### 2.2 E1 — Review Request 1

**Strengths:**

- Opens with a genuine check-in ("how's it going?") before making the ask — builds relationship first
- Social proof framing is excellent: "The honest stories from parents like you are what convinced most of our customers to try. Not marketing. Not claims."

**Issues:**

### 2.3 E2 — Review Request 2 (with 10% Incentive)

**Subject: "One more ask — with a small thank-you"**

**Assessment: GOOD, with one notable issue.**

**Strengths:**
- Opens by thanking those who already submitted — good social grace
- "All of those are real, honest accounts" — inclusive framing that welcomes even partial or uncertain reviews
- The 10% off incentive is well-positioned as a thank-you, not a bribe
- Final line ("This is the last time I'll ask") respects the subscriber's inbox

**Issue: THE REVIEW INCENTIVE CONTRADICTS THE README.**

The README (Flow 18, Purpose section, line 9) explicitly states: **"No incentives offered (no discount stacking)."** But E2 offers **10% off the next order** for leaving a review. This is a direct contradiction between strategy and execution.

The README is the strategic document — it was written to define the flow's parameters. The email implementation ignored this constraint. Either:
- The strategy has changed (review incentives are now allowed), in which case the README must be updated
- The email was built without reference to the README, in which case the incentive must be removed from E2

This is a structural governance issue, not a minor copy tweak. It affects trust in the flow documentation system.

**Recommendations:**
1. **Resolve the incentive contradiction immediately** — either remove the 10% offer from E2 or update README to document the incentive policy change
2. If incentive is kept, document the code structure: unique code per customer vs. static code vs. auto-applied. Currently undocumented
3. If incentive is removed, rewrite E2 to use social proof pressure as the only motivator: "Over 16,000 parents have shared their story. We'd love to hear yours too."
4. Add UTM params as documented in the README (`?utm_source=email&utm_medium=review&utm_campaign=review-request`) — neither email currently includes them in the visible CTA

### 3.2 E1 — Referral Ask

**Assessment: EXCELLENT.**

**Issues:**
- The referral link in the copy (`https://brightkidco.com/referral?ref=CUSTOMER_ID`) uses a placeholder — this needs to be a dynamic Klaviyo merge tag indicating proper integration with a referral platform
- No mention of how the referral is tracked — does the friend enter a code? Click the link? Share via social? This is unclear to the parent
- The mutual benefit ($10 both) is mentioned but not quantified in real terms — "a pair is $34.99, so $10 covers most of the shipping or a good chunk of a single pair" would make it feel more real

No same-day overlap, but the Day 59 (Referral E3) to Day 60 (Review E1) transition is tight. The themes are complementary (community sharing → review sharing), so the overlap is unlikely to cause confusion, but the close spacing may increase unsubscribe risk at this boundary.

**Recommendation:** Monitor unsubscribe rates around Days 59–67. If elevated, consider offsetting one flow by 3–5 days.
