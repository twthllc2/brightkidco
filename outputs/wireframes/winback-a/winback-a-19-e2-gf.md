## 2. Strategy Box

**Big Idea:** Re-engagement through peer solidarity — three different families, three different timelines, one shared experience of pausing and returning. The visual metaphor is community: the reader sees herself reflected in multiple stories, reducing isolation and normalizing the decision to try again.

**Emotional Arc Position:** Position 2 of 3 in Winback A — The reconnection email. E1 established "no guilt, no pressure." E2 answers "has this ever worked for a child like mine?" through the Three-Family Testimonial Pattern. E3 will be the final check-in. This is the pivot from validation to peer proof.

**Component Selection Rationale:**
- HERO-17 (Social Proof Hero): Opens with the three-family hook — "Three families, three different kids, one thing in common." Hero image is lifestyle (nostalgic) per flow signature.
- LETTER-01 (Paragraph Stack): Long-form body copy supports the 450-550 word copy density. Paragraph stack format works for the Whether...Or...If... hook and validation sections.
- SP-09 (UGC Photo Grid): Three-family testimonial pattern — Sarah, Lisa, Maria stories in a grid layout. UGC style matches reconnection mood.
- EDU-10 (How It Works): Mechanism reminder section — Body-Signal Learning Layer explanation. Science-light, symptom-first per GF calibration.
- ProductShowcaseMedium: Mid-level product presentation with 3+3 Bundle offer. Not full showcase (E1 was full), medium level for re-engagement.
- CTA-11 (Permission CTA): "Try again with your 60-day guarantee" — permission-framed, zero urgency per flow signature and GF calibration.
- TRUST-1 (60-Day Seal): Risk reversal — guarantee still stands from original purchase.
- DESDEEP-3 (Card Cascade): Decorative cascade for three-story layout, visual variety.

**Differentiation Note:** GF is the broadest audience — uses three-family testimonial pattern (vs L1's single Pre-K success story, L2's BCBA endorsement, L3's late-trainer story). Hope/realism ratio 50/50. Zero urgency. R1-R6 fully calibrated — no level labels, symptom-only language, age-inclusive, verbal+non-verbal inclusive, timeline-flexible, low-bar success metrics.

## 8. Rendering Notes

### Email Client Considerations
- **Gmail:** Strips CSS animations, classes. All styling must be inline. Max email size ~100KB. Preheader text visible in preview.
- **Outlook Desktop:** No SVG support. Wave dividers degrade to thin rules (#E8DED5). Background colors on tables only. Border-radius not supported on buttons (use VML fallback or accept square corners).
- **Apple Mail:** Full CSS support. SVG renders. Best rendering environment.
- **Yahoo Mail:** Inline CSS required. Border-radius spotty.
- **Thunderbird:** Good CSS support. SVG may not render.

### Mobile Responsive Notes
- **Breakpoint:** 480px
- **Max width:** 600px desktop, full-width mobile
- **CTA button:** Full-width on mobile (spans screen edges minus 30px padding), 52px height
- **Hero image:** Full-width on mobile, maintains aspect ratio
- **Testimonial cards:** Stack vertically on mobile, 100% width
- **Product showcase:** Hero image full-width, offer cards stack vertically
- **Font sizes:** Body 15px mobile, Headlines 22px mobile
- **Spacing:** Section gap 32px mobile (from 40px desktop)

- **Body text minimum:** 15px on mobile (WCAG AA)

- **Alt text:** All images have descriptive alt text
- **Semantic structure:** H1 for hero headline, H2 for section headers, blockquote for testimonials
