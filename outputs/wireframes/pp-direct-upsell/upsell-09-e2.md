## 2. Strategy Box

**Big Idea:** A split-frame visual showing the "two-layer protection" concept — on one side, a child sleeping peacefully in Body-Signal underwear under a cozy duvet; on the other side, a cross-section reveal showing the cotton top, breathable waterproof membrane, and protected mattress beneath. The visual reads like an instructional illustration, honest, educational, and reassuring rather than sales-driven. "The signal happens. The mattress stays safe. Both can coexist."
**Emotional Arc Position:** Utility-focused follow-up to E1's Wash Bag introduction. Transitions from "keep the product working" to "protect your home during the learning phase." Practical problem-solving, not emotional.
**Component Selection Rationale:** HERO-13 (Scientific Hook) opens with the learning-phase reality check. LETTER-01 (Paragraph Stack) carries the validation and social proof in clean blocks. SP-04 (Trust Pills Row) presents the two parent testimonials as trust signals. ED1 (3-Layer Product Diagram) shows the mattress protector's construction. ProductShowcaseFull presents the protector with clear specs. CTA-5 (Product CTA) with moderate urgency matches the utility tone. TRUST-2 (Guarantee Detail Cards) provides structured trust signals. UTIL-07 (Section Header) separates sections cleanly.
**Differentiation Note:** E2 has more animation than E1 — the signal-path dotted line in the cutaway illustration pulses gently, CTA button has magnetic hover, benefit cards fade in on scroll. Gmail/Outlook fallback to static state. The cotton seed easter egg is hidden at 20% opacity. Cross-level: silhouette child figure with no facial features, no age indicators, no gender cues — any parent can project their child onto it.

## 8. Rendering Notes

- **Email client considerations:** Split-frame hero uses standard `<img>` tag — renders in all clients. Cutaway illustration signal-path animation uses CSS `@keyframes` — Apple Mail/iOS only, static fallback for Gmail/Outlook. CTA button magnetic hover uses CSS `transition` — desktop only, static for mobile. Benefit card stagger uses CSS animation with `animation-delay` — desktop only, all cards visible immediately on mobile/Gmail/Outlook. Cotton seed easter egg uses absolute positioning — works in all clients. Diamond dot dividers (◇) use Unicode character — universal support.
- **Mobile responsive notes:** Single-column layout. Split-frame hero stacks to full-width image on screens < 600px. Benefit cards lose rotation, stack vertically with hairline dividers. CTA buttons go full-width on mobile. Product image shrinks to full-width crop focused on cotton texture. Section padding reduces to 16px on mobile. Minimum 44px tap targets.
- **Accessibility notes:** All images have descriptive alt text. Color contrast: body text #5C5A55 on #FFFFFF passes WCAG AA (~5.5:1). CTA white on #039902 emerald passes WCAG AA (~4.8:1). Testimonial text on #FBF7F1 passes WCAG AA. Price callout #039902 on #FFFFFF passes WCAG AA (~4.8:1). Footer text #8A9B9D on #FBF7F1 passes WCAG AA (~3.5:1 — meets AA for large text).
- **Special instructions:** Cross-level email — no level labels anywhere. Child figure is silhouette-only (no facial features, no age indicators, no gender cues) — any parent can project their child onto it. Bedroom setting is generic — no toys suggesting specific age, no wall decorations suggesting verbal/non-verbal status, no school-related items. Color palette is intentionally neutral-warm — no bright "kid" colors, no Pre-K visual cues, no clinical elements. Typography sizing follows "maximum breathing room" standard (L3's preferred pacing) so L1 readers aren't overwhelmed and L3 readers aren'trushed. CTA is medium prominence — not tiny (would feel dismissive to L1-L2), not oversized (would pressure L3). Cotton seed easter egg is the only level-calibrated element, hidden at 20% opacity so it doesn't break cross-level consistency. No timeline promises — "learning phase" framing is open-ended. No success metrics promised — utility upsell, not outcome email.
