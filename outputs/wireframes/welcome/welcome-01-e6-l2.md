## 2. Strategy Box

**Big Idea:** The signal your child needs is the one pull-ups take away — a counterintuitive reveal flipping pull-ups from necessary tool to unintentional barrier. Dryness is not the goal, sensation is.

**Emotional Arc Position:** E6 sits at the "Learning Objections" stage (Day 14) in the Welcome Flow. The L2 parent (Lisa) has received E1-E5 covering hope, mechanism, decision, wisdom, and accepting mechanism specifics. E6 is the moment where the most common objection — "but pull-ups work" — is reframed. The emotional arc: Surprise (pull-ups are the problem) → Mechanism reveal (wetness signal is the missing feedback) → Guilt-free awareness (you're not wrong, the product is) → Cautious motivation (60-day trial by your judgment).

**Component Selection Rationale:** HERO-7 (Counter-Intuitive Truth) delivers the pull-up reframe as the hook. LETTER-09 (Double Column) provides the two-column mechanism layout for the pull-up vs BKC comparison. SP-02 (Expert Endorsement) adds credibility for the BCBA-aware L2 segment. EDU-10 (How It Works) presents the 3-layer Body-Signal Learning System. ProductShowcaseFull (Hero product moment) is the full product card with CTA. CTA-5 (Product CTA) drives the conversion. TRUST-2 (Guarantee Detail Cards) provides the 60-day guarantee breakdown. 20.2 (Dot Pattern Divider) separates sections with the L2-appropriate dot pattern.

**Differentiation Note:** Compared to E6-L1 (which uses HERO-19 Timeline Shock, EDU-4 Myth vs Fact, warmer colors, and moderate urgency), the L2 version is clinical-honest and BCBA-referencing throughout. Compared to E6-L3 (which uses HERO-9 Whisper Tone, ProductShowcaseSoft, zero urgency, and a single testimonial), the L2 version has full product showcase, 3 social proof quotes, and a direct but permission-framed CTA button.

## 8. Rendering Notes

### Email Client Considerations
- **Outlook:** Fallback fonts for serif italic heading (Times New Roman → Georgia → serif). Bulletproof CTA buttons (VML fallback for Outlook). Two-column layout uses table-based markup with MSO conditional comments.
- **Gmail:** Max email width 420px. All styles inline. Background images require VML for Outlook.
- **Apple Mail:** Full CSS support including animations (fade-in, staggered layer reveal).
- **Samsung Mail:** Test two-column stacking at 480px breakpoint.

### Mobile Responsive Notes
- **Breakpoint:** 480px — two-column layouts stack vertically
- **Heading sizes:** 28px → 22px on mobile
- **Body text:** 15px → 14px on mobile, line-height 1.7 maintained
- **Product card:** Full width, sub-grid stacks vertically
- **Hero image:** Full width, reduced height (200px → 160px)
- **CTA buttons:** Full width on mobile, padding maintained
- **L2-specific:** Copy density is long — ensure adequate vertical rhythm. No content is hidden on mobile; everything scrolls.

### Accessibility Notes
- **Alt text:** All images have descriptive alt text. Hero image comparison clearly described.
- **Color contrast:** B.ink (#1F2D2F) on #FAFAF8 = 12.5:1 ratio (exceeds WCAG AAA). B.soft (#4A6568) on #FAFAF8 = 5.2:1 ratio (exceeds WCAG AA).
- **Font sizes:** Minimum 10.5px (eyebrows), all body text 15px+. No text below 10px.
- **Link text:** CTAs have descriptive text ("Try the 3+3 Bundle", "Shop the pants"). No "click here" patterns.
- **Semantic HTML:** Use h1 for main heading, h2 for section headings, blockquote for pull quotes.
