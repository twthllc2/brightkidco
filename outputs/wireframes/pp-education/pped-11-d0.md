## 2. Strategy Box

**Big Idea:** Week 1 is not about the toilet. It's about the nervous system learning a new language.

**Emotional Arc Position:** First position in PP-Education Flow — the immediate post-purchase email. Parent has just bought the underwear and is in "excited entry" mode. This email must reset expectations immediately: from "I bought the underwear → my child should start using the potty" to "I bought the underwear → my child's nervous system needs time to register a new signal."

**Component Selection Rationale:**
- HERO-16 (Mechanism Tease) — leads with the mechanism framing, not a lifestyle hero. The parent needs to understand the product works passively before anything else.
- LETTER-01 (Paragraph Stack) — long-form letter structure sustains the educational, research-grounded tone. Multiple paragraphs let the copy build from validation → instruction → mechanism.
- SP-08 (Stacked Mini-Reviews) — three-family testimonial pattern, stacked vertically. Each testimonial maps to a different level without labeling. The Easter egg: hidden calibration table in the testimonial selection.
- EDU-8 (Mechanism Diagram) — visual mechanism deep dive. The BSL Layer explanation needs a clear diagram showing the 30-60 second feedback window.
- ProductShowcaseSoft — gentle product presence, not a hard sell. The product has already arrived; this is education, not conversion.
- CTA-2 (Soft Link) — permission-based CTA. "Keep a progress log →" not "Buy now." The parent already owns the product.
- TRUST-1 (60-Day Seal) — lightweight trust anchor. Not the full guarantee detail cards (that's for upsell emails).
- UTIL-01 (Gradient Band) — section dividers between major blocks. B.teal → B.green gradient.

**Differentiation Note:** D0 is unique in the PP-Education flow because it's the only email that must simultaneously validate the purchase decision AND reset expectations downward. Every other email in the flow assumes the parent has already been wearing the product for days/weeks. D0 is the only one that addresses the "I just bought this" mindset. The "Whether...Or...If..." opener is the structural innovation — it encodes 3 dimensions (age, communication, journey stage) in one sentence, catching all 4 levels + all 5 GF sub-groups.

Section spacing compressed from 48px to 30px on mobile. Body text minimum 16px on mobile. Touch targets minimum 44x44px.
- **Accessibility notes:** All images require meaningful alt text. Color is never the sole carrier of information. Win checkmarks use ✓ character + text, not color alone. CTA button text is sufficient without color context. Testimonial em-dash "—" is semantic, not decorative.

The "49%" stat highlight uses background-color only (no CSS animation in email).
