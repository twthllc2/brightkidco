## 0. HOW TO USE THIS DOCUMENT

This blueprint treats every L2 email as a structural artifact, not just copy. The **L2 axiom** governs everything: *marketing polish triggers skepticism; mechanism specificity converts her into a scientific evaluator.* So we build structure that reads as **science-led**, not **promo-led** — by varying 9 dimensions deliberately to prevent pattern recognition, by anchoring every L2 email in the **6-element formula** (Demand §23), and by treating the **L2 unlock sentence** as the gravitational center.

### 0.1 The 9 variance dimensions

Per the `email-marketing-structure` skill §9.2, every email is mapped across:

| # | Dimension | L2 Range / Anchor |
|---|---|---|
| 1 | **Section count + names** | 3–8 sections; science callout boxes, micro-moments, BCBA anchors named explicitly |
| 2 | **Section pattern (A–H)** | Pattern C (Problem-Solution), D (Social Proof First), F (Education-First), H (Minimalist) dominate; Pattern G (Urgency) is **NEVER** for L2 |
| 3 | **Image density** | LOW (0–3) for mechanism/cart/checkout; MEDIUM (2–4) for story/mechanism; 60:40 text-to-image |
| 4 | **Color temperature** | Cool (mint #DBFFCD) dominant — the L2 visual signature; warm (cream #FBF7EB) for transitional/retention; warm for Mary S. founder story |
| 5 | **Email length** | 350–900 words; mechanism-heavy (Welcome E2/E5/E7) = 700–900; transactional = 150–300 |
| 6 | **Argument structure** | Mechanism-first, story-led, or data-led; **never direct/urgency** |
| 7 | **Visual rhythm** | Open (44px section gaps, generous whitespace around CTAs); tighter (24–32px) only in mechanism blocks |
| 8 | **CTA position** | Late or text-link-only in Welcome; double-tap permitted in PP-Mid/Replenishment; NEVER early CTA in Welcome (reads as salesy) |
| 9 | **Subject line type** | Observation, question, mechanism tease, peer quote, BCBA-respect anchor; **NEVER** exclamation/urgency/AI-ism |

### 0.2 L2 visual signature — what carries across every email

- **Mint-bordered science callout boxes** (avatar §9) around every statistic, citation, and mechanism explanation. This is the L2 visual signature; its visual message is *"this is the science part. Slow down and read it."*
- **Cream card backgrounds** (#FBF7EB) for testimonial cards and product blocks
- **Muted green CTA** (desaturated #039902 hue), 48px min height, **NOT full-width**, 12–16px border-radius
- **Photography dominates** for L2 (real kids 5–7 in real sensory contexts); illustration reserved for mechanism explainers
- **No countdown timers, no "Quick," no exclamation cascade, no "Quick results"**

### 0.3 L2 sign-off progression (locked)

| Position | Sign-off | Variant |
|---|---|---|
| Welcome E1–E2 | "Talk soon," | SHORT |
| Welcome E3–E5 | "With understanding," | MEDIUM |
| Welcome E6+ | "With care, Lena." | LONG / DEEPEST |

# VARIANCE LEDGER — L2 SEQUENCE

> Per the `email-marketing-structure` skill §9.2: for each email N, list the previous 3 emails' dimensions, count differences across the 9 axes, and flag any consecutive email pair where fewer than 6/9 dimensions differ. **Threshold rule: < 6/9 differ = desensitization risk.** Recommend structural fixes where flagged.

**Methodology.** For each "differ" check we compare across the 9 dimensions using the value labels assigned in the per-email tables above. A dimension "differs" if the letter/numeric value (e.g., Pattern C vs Pattern H; Cool vs Warm; 4 sections vs 8 sections) changes between the two adjacent emails. **Section count** differs when the integer changes; **Pattern** differs when the letter (A–H) changes; **Image density** differs when the bin (0 / 1 / 2 / 3 / 3+) changes; **Color** differs when the temperature label (Cool / Warm / Neutral) changes; **Length** differs when the bin (XS / S / M / L / XL) changes; **Argument** differs when the type label (Mirror / Mechanism / Testimonial / Data / Story / etc.) changes; **Rhythm** differs when the spacing category (Tight / Mixed / Open) changes; **CTA position** differs when the position category (Early / Mid / Late / Double-tap / Text-link) changes; **Subject type** differs when the subject category (Observation / Question / Mechanism / Peer / Data / Lena / Permission / etc.) changes.

**Length bins:** XS ≤ 150 words; S = 150–300; M = 300–500; L = 500–800; XL = 800+.
