# STAGE 3 — HTML BUILD (Complete Plan)

**Date:** June 29, 2026 | **Status:** Ready for execution

> **CORE RULE:** One subagent = one email design. No scripts. No batching. No HTML-generating functions. Every decision has already been made by Stages 1 and 2. Stage 3 is ASSEMBLY — turning the blueprint into a real email.
>
> **THE BRAND-IDEA:** Giuliano's JSX primitives define the visual language. Stage 1 wireframes define the layout. Stage 2 picks define the specific components. Stage 3 maps them together with no CSS invention.

## 0. WHAT STAGE 3 IS

Stage 3 turns each of the 125 fully-specified email blueprints into production HTML. For every email, a single subagent reads:
- The copy file (original copy text)
- The wireframe (ASCII layout blueprint with ROLES per section)
- The Stage 2 picks (specific component ID + image filename for each ROLE)
- The WRAP tags (how each image relates to its component)
- The brand spec (token.js + primitives.jsx — the exact visual language)
- The locked header and footer (extracted from Giuliano's design)
- The level calibration rules (from the journey master)

And produces ONE HTML file. 125 subagent jobs. 125 HTML files.

**Stage 3 does NOT:**
- Pick components (Stage 2 did that)
- Decide layout (Stage 1 did that)
- Change copy text
- Change images
- Use scripts for design decisions

### 1.6 Other Invariants

- **Mobile breakpoint:** 480px. All emails must include a `@media only screen and (max-width:480px)` block that:
  - Sets body text to 16px minimum
  - Sets headline to 22px minimum
  - Makes CTA buttons full-width
  - Collapses any multi-element rows to single-column
  - Reduces section padding from 22px to 16px
- **Google Fonts link:** REQUIRED in `<head>`:
  ```html
  <link href="https://fonts.googleapis.com/css2?family=Questrial&family=Fraunces:ital,wght@0,400;0,700;1,400&family=Caveat:wght@400;500;700&display=swap" rel="stylesheet">
  ```
  This is a web font request. Use `display=swap` so text renders immediately with fallback fonts while web fonts load.
- **Outlook fallback fonts — ALWAYS include:**
  - Body: `font-family: 'Questrial', Arial, Helvetica, sans-serif;`
  - Display/headline: `font-family: 'Fraunces', Georgia, 'Times New Roman', serif;`
  - Handwritten (Lena sig only): `font-family: 'Caveat', 'Comic Sans MS', cursive;`
  Never use a web font without the fallback stack. Outlook ignores web fonts entirely.
- **Gmail is the primary target.** Gmail strips `<style>` blocks (put ALL CSS inline). Gmail ignores `max-width` on the outer container (use `width:600px` on the inner table). Gmail clips emails over 102KB at a random point in the body — keep total under 100KB to be safe.
- Preheader div: HIDDEN — `<div style="display:none; max-height:0; overflow:hidden; mso-hide:all;">{preheader text}</div>`
- Preheader text must COMPLEMENT the subject line, not repeat it (per demands §11.2)
- Subject line ≤ 50 characters ideally
- CTA button min 48px height (tap-friendly per demands §15.5)
- MSO conditionals for Outlook (include `<!--[if mso]>` wrappers around Outlook-specific fixes)
- Hidden plain-text fallback for transactional (per demands §17.7)
- Max total size <102KB (Gmail clip threshold). If over 100KB after inlining images:
  - FIRST: resize hero/product images to lower quality (JPEG quality 70-80%)
  - SECOND: reduce to max 500px width instead of 600px
  - THIRD: if still over, fall back to linking the largest image externally (absolute last resort)
- HOW TO DETERMINE EMAIL POSITION (for sign-off variant):
  - Parse the email_id. Example: `welcome-01-e1-l1` → flow="welcome", email_position="e1", level="l1"
  - The email_position ("e1", "e2", "e3", etc.) determines the sign-off variant per Section 1.5
  - If the email_id doesn't follow this pattern, check the wireframe's section count — lighter emails typically use SHORT, deeper ones use LONG

## 2. THE BODY SECTIONS — HOW TO BUILD THEM

### 2.1 The Universal Section Stack

Every section in the body follows this stack (from Stage 1's wireframe rule):

```
┌──────────────────────────────┐
│ HEADLINE (text only)         │  ← The section's heading
├──────────────────────────────┤
│ SUBHEADLINE (if needed)      │  ← Optional context line
├──────────────────────────────┤
│ IMAGE OR COMPONENT           │  ← The visual element goes HERE (below headline)
├──────────────────────────────┤
│ CTA (if section has one)     │  ← Optional CTA button/link
└──────────────────────────────┘
         ↓ gap (40-56px) ↓
```

NEVER side-by-side. NEVER two-column. NEVER image-LEFT text-RIGHT. The headline goes FIRST, then the image or component. Always. (per Ayoub's Stage 1 RULE)

### 2.3 The WRAP Tag Rule

For each section, the WRAP tag determines how the image relates to the component:

- **WRAP: image-inside-component** → The image is INSIDE the component's HTML. E.g., a hero image inside the TitleBlock, a product photo inside the Product Card, a quote photo inside the Pull-Quote callout.
- **WRAP: image-standalone** → The image sits in its OWN `<td>` between sections (not inside any component). It's a standalone visual element.
- **WRAP: component-text-only** → No image. The component is purely text.
- **WRAP: image-required-component** → A BUG flagged from Stage 2. The component needs an image but none was picked. The subagent MUST still render the component (it's in the picks) but should note the issue. Do NOT skip the component.

### 2.4 Image Handling

Every image picked by Stage 2 must be:
1. **Base64-inlined** as a data URI (NEVER linked to external URL — emails get forwarded, CORS blocks external images)
2. **Resized** to max 600px width before encoding (Gmail clips at 102KB total, the logo alone is 200KB base64)
3. **Alt-text** applied per the Stage 2 pick's specified alt text and V23 level spec (L1 = active/forward-looking, L2 = calm/sensory-aware, L3 = reflective/dignity)
4. **Image count per email** must match what Stage 2 specified (4-6 images HARD minimum). If the email has fewer than 4 images due to WRAP text-only sections, Stage 2's picks should have already covered the shortfall with image-standalone picks.

## 3. THE LEVEL CALIBRATION RULES

**L1 (Sarah) — ~35% of audience:**

- **Spacing:** Tighter, 40px section gaps

- **Sign-off:** "Talk soon," (E1-E2) → "With understanding," (E3-E5) → "With care," (E6+)

**L2 (Lisa) — ~40% of audience:**
- **Color energy:** Muted greens, desaturated mint tones, less yellow
- **Spacing:** Balanced, 44px gaps
- **CTA:** Muted green, 48px height, lower aggression
- **Imagery:** Kids 5-7, sensory context, BCBA-respectful, never Pre-K imagery
- **Subject line:** Mechanism tease, data-led ("Why sticker charts didn't work — neurologically")
- **Forbidden:** "Pre-K is coming," "Easy potty training," BCBA criticism, "Just like other kids"
- **6-element L2 formula:** Symptom recognition + mechanism specificity + BCBA acknowledgment + realistic timelines (8-12 wks) + 60-day guarantee + permission-not-to-buy. Every L2 email must include all 6.
- **Sign-off:** Same progression as L1

**L3 (Maria) — ~25% of audience:**
- **Color energy:** Cream/earth tones ONLY — #FAF8F5, #FBF7EB, #F5F2ED, #D4C5B2, #C4A88B
- **ZERO yellow** (#F5C84B NEVER used), **ZERO Brand-Green** filled CTA buttons
- **Spacing:** WIDEST, 56px gaps, generous breathing room
- **CTA:** Text-link ONLY or very small muted button. NEVER full-width green. NEVER urgency.
- **Imagery:** Kids 6-10+, quiet warmth, dignity-centered, reflective. NEVER Pre-K, NEVER celebration.
- **Subject line:** Validation or data only. NEVER observation hooks that imply "you're doing it wrong."
- **Forbidden:** "Don't give up!", "Fix", "Cure", "Independence!", any urgency, any cheerleading, "less than a coffee" math reframe, adult diaper sales language, specific carrier names, prominent shipping cost.
- **Primary offer:** 1 Pair $34.99 (NOT the bundle). Permission-not-to-buy in EVERY mail.
- **Sign-off:** Same progression as L1, but closing body text adapted to L3 dignity-first language.

**GF (General Fallback) — ~50% real volume:**
- **Color energy:** Balanced mid-saturation, Brand-Green #039902, Mint #DBFFCD, Cream #FBF7EB
- **Spacing:** Balanced, 44px gaps
- **CTA:** Medium-low aggression, balanced
- **Imagery:** Cross-level, universal, symptom-first, never level-labeled
- **Subject line:** "Whether...Or...If..." pattern, symptom-first, age range 3-10+
- **R1-R6 strict:** NO autism labels, NO ASD, NO "Level 1/2/3" in component copy. V16 symptom language only.
- **Migration anchors:** Embedded in E2 (L1-style), E4 (L2-style), E5 (L3-style), E8 (Assessment link)
- **Sign-off:** Same progression as L1
