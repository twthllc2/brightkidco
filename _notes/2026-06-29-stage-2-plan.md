# STAGE 2 — ASSET SELECTION (CORRECTED June 29, 2026)

> **CORE RULE:** One subagent = one section of one email. That's it. No HTML, no CSS, no scripts, no design decisions. Stage 2 = pick the right component and image for the ROLE that Stage 1's wireframe already specified.
>
> **If you're tempted to write HTML, you are in the wrong stage. Stage 2 is selection. HTML building is Stage 3.**

---

## 0. WHY THIS EXISTS

Stage 1 produced 125 wireframes. Each wireframe has, for every section, a ROLE name (e.g., `hero-image-lifestyle`, `callout-quote-highlight`, `stat-highlight-box`, `cta-button-full-width`, `signoff-lena-short`) plus an INTENT (what the section should communicate) plus an EXAMPLE_PICK (what kind of asset would fit).

Stage 2 picks the SPECIFIC component ID from the 949-catalog and the SPECIFIC image filename from the 36-library that fills each ROLE slot — while respecting the 8x component cap and 25x image cap.

Stage 2 also writes the picks BACK to the copy file (so Stage 3/4 can read them) and updates the shared trackers (so subsequent Stage 2 subagents don't pick the same over-used assets).

Stage 2 does NOT touch HTML, CSS, or email design. It is a reasoning + bookkeeping job. Every step is visible and auditable.

## 3. JOB STRUCTURE (per email)

For each of 125 emails, Stage 2 dispatches `N` subagents where N = number of distinct ROLE sections in the wireframe (typically 3-8 per email).

**For a 5-section email: 5 subagents. For an 8-section email: 8 subagents. For a 3-section email: 3 subagents.**

Plus 1 tracker-update subagent per email that reads all the picks, validates no caps were violated, and writes the consolidated tracker increments.

**Total Stage 2 subagent jobs: ~625 section picks + 125 tracker updates = ~750 subagent jobs.**

## 4. THE SUBAGENT PROMPT (per section, repeated 625+ times)

Every section subagent gets THIS prompt (with only the email_id, flow, section_index, and role_slot changing):

```
You are picking the right asset for ONE section of ONE BKC email.

INPUTS:
1. Copy file: /root/projects/brightkidco/outputs/copy/{flow}/{filename}.md
   - Read the WIREFRAME section (lines from "## WIREFRAME — built" to "END OF WIREFRAME")
   - Find the ROLE for section {N} (the section you're picking for)
2. Locked profile: /root/projects/brightkidco/_notes/locked-profile.json — find your email_id
3. Journey master: /root/projects/brightkidco/_notes/journey-{level}-MASTER.md — read the section about visual calibration for your level
4. Cross-level sync: /root/projects/brightkidco/_notes/journey-cross-level-sync.md — check for known conflicts
5. Component catalog: /root/projects/brightkidco/outputs/email-assets/component-catalog.json — 949 components, filter by `position` matching the ROLE and `audience` matching the level
6. Component showcase: /root/projects/brightkidco/outputs/email-assets/showcase-artifacts-final.html — visual reference (open the file, find the component, see what it looks like)
7. Image library: /root/projects/brightkidco/outputs/email-assets/product-photos/ — 36 images across 5 subfolders
8. Component-usage tracker: /root/projects/brightkidco/_notes/component-usage.json — read counts, skip anything at 8
9. Image-usage tracker: /root/projects/brightkidco/_notes/image-usage.json — read counts, skip anything at 25

TASK:
1. Read the wireframe's ROLE slot for section {N} (e.g., `callout-quote-highlight`)
2. Read the INTENT — what this section should communicate
3. Read the level-specific calibration rules from the journey master
4. Browse the component catalog: filter by ROLE-matching position and audience
5. Open the showcase HTML to VISUALLY inspect the top 3-5 candidates (they LOOK different even if labels match)
6. Skip any component already at 8 uses
7. Skip any component already used in this same email
8. Skip any component that violates level-specific rules (e.g., NEVER Yellow for L3, NEVER full-width CTA for L3)
9. PICK the best-fit component. Document WHY in 1-2 sentences (visual match, audience match, level match)
10. If the section needs an image: browse /root/projects/brightkidco/outputs/email-assets/product-photos/, find the best match for the INTENT, skip anything at 25 uses, skip duplicates in this email, pick
11. Document alt text for the image per L1/L2/L3/GF V23 rules
12. APPEND the structured record to the copy file under a new heading "## STAGE 2 PICKS"
13. UPDATE the component-usage.json and image-usage.json with your picks (increment the counts)

VARIANCE CHECK:
- If this is NOT the first section of this email, check what was picked for the previous section
- Avoid picking the SAME component ID for two adjacent sections in the same email
- If variance is at risk, prefer a different component even if slightly less optimal

CONSTRAINTS:
- Component cap: 8 uses across all 125 emails (HARD STOP at 8)
- Image cap: 25 uses across all 125 emails (HARD STOP at 25)
- Level rules per journey-MASTER (L3 ZERO yellow, etc.)
- Do NOT use teal #2BAEB4 (System A — explicitly banned)
- Do NOT introduce HTML, CSS, or any email markup
- Do NOT change the copy text — the wireframe's INTENT is what you match against, not what you replace

REPORT BACK:
"Email {email_id} section {N} picked. Component: {ID} (now N/8 used). Image: {filename} (now N/25 used). WHY: {1-2 sentence justification}."
```

## 5. CONSTRAINTS (NON-NEGOTIABLE)

### 5.1 Caps
- **Component cap: 8 uses** across all 125 emails. If a component is at 8, skip it. HARD STOP.
- **Image cap: 25 uses** across all 125 emails. If an image is at 25, skip it. HARD STOP.
- These caps are tracked in `component-usage.json` and `image-usage.json` — both READ and UPDATED by every subagent.

### 5.2 Level-specific rules
Read from the journey-MASTER per level:
- **L1 (Sarah)**: bright green CTAs, yellow stat accents, Pre-K imagery, full saturation colors
- **L2 (Lisa)**: muted greens, BCBA acknowledgment, science callout boxes, 6-element formula compliance
- **L3 (Maria)**: ZERO yellow, ZERO urgency, cream/earth tones only, text-link or muted button CTAs, 56px spacing, dignity-first, NEVER "Don't give up!" / "Fix" / "Cure", 1 Pair $34.99 primary offer, NEVER math reframe, NEVER adult diaper sales language
- **GF (General Fallback)**: R1-R6 strict, NO autism/ASD/Level 1/2/3 labels in body, V16 symptom language only, sub-group accommodation, migration anchors in E2/E4/E5/E8

### 5.3 Cross-level conflicts (from journey-cross-level-sync.md)
The 8 HIGH-severity conflicts identified in Stage 0.5 must be honored. Each subagent reads the sync report and applies relevant conflicts.

### 5.4 No teal
Giuliano's System A used teal #2BAEB4. System B doesn't. NEVER pick any component whose primary color is teal. NEVER pick any image that prominently features teal branding.

### 5.5 One subagent per section
Each subagent handles ONE section. A 5-section email gets 5 subagents (run sequentially OR in parallel depending on resources). NO single subagent picks for the whole email — context dilution risk.
